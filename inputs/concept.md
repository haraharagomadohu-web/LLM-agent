# VibeFlow: AI主導開発のためのビジュアル設計ブリッジ

## 1. ビジョン
**「AIのひらめきに、人間のこだわりを。」**
VibeFlowは、AI（Cursor, Claude, GPT等）が提案したロジックを非エンジニアが直感的に修正・指示し、100%意図通りのアプリ開発を実現するための「対話型設計プラットフォーム」です。

## 2. 解決したい課題
- **AIの実装が「ブラックボックス」化する**: プロンプトだけで指示を出すと、内部ロジックがAI任せになり、予期せぬ挙動や修正困難なコードが生成される。
- **指示の「再現性」と「正確性」の欠如**: 言葉だけでは複雑な条件分岐やデータフローを伝えきれず、何度もやり直し（リテイク）が発生する。
- **YAMLベースの対話の限界**: AIと構造的な設計図（YAML）で対話するのが効率的だが、人間が直接大量のYAMLを書いたり読んだりするのは苦痛である。

## 3. 解決策：Vibe-Feedback ループ
人間がコードを書くのではなく、**「AIが書いた設計図（YAML）をGUIで直し、それをまたAIに渡す」**という新しい開発サイクルを提唱します。

1.  **AI提案**: AIに「〇〇な機能のフローをYAMLで出して」と依頼。
2.  **GUI可視化**: 出力されたYAMLをVibeFlowに配置。複雑な構造が「ブロック線図」として一瞬で可視化される。
3.  **人間による調整**: GUI上で「ここはAじゃなくてBに通したい」「この処理の後にこのパラメータを追加して」と直感的に修正。
4.  **AI実装**: 正確に書き換えられたYAMLをAIに返し、「この通りに実装して」と指示。

## 4. 主な機能
- **YAML ↔ Blocks 双方向変換**: AIの出力を即座にビジュアル化し、ビジュアル操作を即座にYAML化。
- **AIフレンドリー・スキーマ**: AIが理解しやすく、かつ人間が編集しやすい独自のYAMLフォーマット。
- **こだわりパラメータ設定**: 各ブロックの詳細な挙動（プロンプトへの追記内容など）をプロパティパネルで細かく指定。
- **テンプレートライブラリ**: 既存のワークフローツール（n8n等）をリスペクトした、標準的な処理ブロック群。

## 5. 本アプリの存在意義（コア・バリュー）
### ① 非エンジニアへのパワー付与
コードが書けなくても、論理構造（アルゴリズム）さえ組み立てられれば、エンジニアと同等かそれ以上の精度でAIに実装をさせることができます。

### ② Vibe Codingの「再現性」の担保
「なんとなく動く」ではなく、「設計図通りに動く」状態を作るため、プロジェクトの資産として設計図が残ります。

### ③ 開発速度の極大化
複雑なロジックをプロンプトで説明する時間をゼロにし、図を動かすだけで完璧な指示書が完成します。

## 6. 技術リファレンス・構築方針
### ① UI/GUIの構築
- **ベース（パクリ元）**: GitHubで公開されている **[n8n](https://github.com/n8n-io/n8n) のGUIデザインおよびレイアウト**。
    - ワークフローツールのデファクトスタンダードである n8n の「3カラム構成（Nodes / Canvas / Settings）」を、リポジトリを参考にそのまま継承・活用します。


### ② 開発スタック
- **エンジン**: React Flow による高速・美麗なレンダリング
- **ターゲット**: Cursor, Claude 等のAIエージェントを使いこなす全ての開発者・クリエイター
- **シリアライズ**: YAML (PyYAML/js-yaml 互換フォーマット)
    - **全単射（Bijective）な変換**: GUI上の全ての要素（配置、プロパティ、接続関係）はYAMLと1対1で対応し、YAMLからGUIに戻した際も情報は一切欠損しない「完全な双方向変換」を保証します。これにより、AIと人間が全く同じ情報を共有できるようにします。
